# **НАЗНАЧЕНИЕ ПРОГРАММЫ**
## **Функциональное назначение**

Проект направлен на создание мобильного приложения для поиска единомышленников на основе общих интересов и фандомной принадлежности. Основная цель — предоставить пользователям интеллектуальную систему подбора потенциальных друзей по интересам, объединяющую механизмы рекомендаций и общения. Приложение обеспечивает взаимодействие пользователей посредством обмена контентом, взаимных оценок анкет и личной переписки, а также формирует персонализированные рекомендации на основе указанных предпочтений.

## **Эксплуатационное назначение**

Программа предназначена для пользователей, заинтересованных в расширении круга общения на основе тематических интересов.

Благодаря проекту пользователи смогут искать людей с похожими интересами и увлечениями, устанавливать социальные связи, вести общение посредством чатов, публикаций и обмена тематическими постами, формировать сообщества на основе фандомов и творческих направлений.

## **Состав функций**

Приложение «FandoMatch» предоставляет следующие функции:

1. Авторизация и регистрация пользователей с валидацией введенных данных с возможностью восстановления пароля

2. Формирование персонализированной ленты рекомендаций пользователей на основе совпадения интересов

3. Настройка пользователем фильтров рекомендаций

4. Просмотр карточек пользователей в формате стека с возможностью свайпов и оценки предлагаемых профилей и созданием мэтча при обоюдном лайке

5. Просмотр профиля другого пользователя с отображением информации, списка фандомов и публикаций

6. Управление собственным профилем: просмотр, редактирование данных, добавление и удаление фандомов, смена пароля, удаление аккаунта

7. Создание публикаций с возможностью добавления текста, изображений, видео и тегов фандомов

8. Просмотр ленты постов пользователей с возможностью перехода к отдельному посту

9. Добавление комментариев и лайков к посту

10. Обмен в личных чатах сообщениями, могущими содержать текст, изображения и видео

11. Просмотр изображений и видео в полноэкранном режиме с возможностью пролистывания и сохранения их на устройство

12. Получение push-уведомлений о новых сообщениях и мэтчах

13. Обработка сетевых ошибок и отображение соответствующих уведомлений пользователю

# **УСЛОВИЯ ВЫПОЛНЕНИЯ ПРОГРАММЫ**

## **Минимальный состав аппаратных средств**

Для полноценного функционирования программы требуются следующие программные средства: ОС Android 8.0 или выше.

# **ВЫПОЛНЕНИЕ ПРОГРАММЫ**

##  **Установка программы**

Установка мобильного приложения происходит с помощью загрузки .apk файла. Файл может быть загружен по ссылке, находящейся в начале данного README.md файла. Загрузка является свободной и не требует от пользователя каких-либо дополнительных действий.  
Для работы с приложением в режиме отладки следует клонировать репозиторий по той же ссылке и открыть его в IDE Android Studio.

## **Запуск программы**

Для запуска приложения достаточно нажать на иконку приложения на устройстве пользователя.

Для запуска приложения в режиме отладки через IDE Android Studio следует открыть склонированный проект в Android Studio и запустить его способом, указанным в IDE.

## **Выполнение программы**

### **Регистрация**

Для создания новой учетной записи пользователь должен заполнить форму регистрации, состоящую из нескольких последовательных экранов и включающую следующие поля:

1. Имя (2-30 символов, может содержать только буквы, пробелы и апострофы)

2. Логин (5-15 символов, только латинские буквы, цифры, дефисы, нижние подчеркивания; логин должен быть уникальным в системе)

3. Электронная почта (рабочий почтовый адрес)

4. Код подтверждения, отправленный на указанную ранее почту

5. Дата рождения (минимальный допустимый возраст пользователя составляет 16 лет)

6. Пол

7. Аватар профиля

8. Пароль (не менее 8 символов, содержащий латинские буквы, цифры и специальные символы)

9. Повтор пароля (должен совпадать с паролем)

Пользователь также обязан подтвердить согласие с правилами сервиса посредством установки соответствующего флага.

При корректном заполнении всех полей после нажатия кнопки регистрации создается учетная запись, автоматически формируется пользовательская сессия, и осуществляется переход на главную страницу приложения. При наличии ошибок (некорректный формат данных, несовпадение паролей, занятый логин) выводятся соответствующие сообщения.

Проверки полей ввода имени, электронной почты, кода подтверждения и пароля одинаковы на всех экранах приложения, поэтому они будут опускаться при описании следующих экранов.

### **Авторизация**

При первичном запуске приложения пользователю показывается начальная страница с кнопками «Зарегистрироваться» и «Войти». Если у пользователя уже существует учетная запись, ему необходимо перейти на экран авторизации и ввести логин и пароль, указанные при регистрации. После нажатия кнопки «Войти» выполняется проверка корректности введенных данных на сервере.

В случае успешной аутентификации создается пользовательская сессия, сохраняется токен авторизации, и пользователь перенаправляется на главную страницу приложения. При вводе некорректного логина или пароля отображается сообщение об ошибке. При отсутствии подключения к сети выводится уведомление о невозможности соединения с сервером.

Также пользователь может перейти на страницу восстановления пароля, ввести на ней почту и отправленный на эту почту код подтверждения, чтобы сменить пароль своей учетной записи. После успешной смены пароля он будет перенаправлен обратно на страницу авторизации.

### **Поиск знакомств**

Главная страница приложения предназначена для поиска знакомств и представляет собой стек карточек пользователей. Каждая такая карточка содержит основную информацию о пользователе: его аватар, имя, возраст, город, пол, описание профиля, список фандомов, процент совместимости с текущим пользователем, а также кнопки лайка и дизлайка. Пользователь может смахнуть карточку вправо или нажать кнопку лайка, если он заинтересован в знакомстве с данным человеком, и смахнуть карточку влево или нажать кнопку дизлайка в противоположном случае. При нажатии на карточку осуществляется переход на страницу соответствующего ей профиля пользователя. Если два пользователя поставили лайк друг другу, между ними формируется взаимное совпадение, или же мэтч, они добавляются друг к другу в друзья, и на их устройства отправляется push-уведомление с информацией о мэтче.

Пользователь может фильтровать предлагаемые ему профили по полу, возрастному диапазону, фандомам, категориям фандомов и городу. Для этого он должен настроить и сохранить фильтры на экране фильтров, там же фильтры можно сбросить в их изначальное состояние. При отсутствии пользователей, подходящих по заданным фильтрам, на главной странице отображается соответствующее информационное сообщение.


### **Профиль пользователя**

На экране профиля отображается личная информация пользователя (имя, возраст, пол, город, описание профиля, список фандомов, аватар и фон профиля) и его посты.

При просмотре профиля другого пользователя в верхней панели отображаются кнопки взаимодействия с ним: лайк и дизлайк для еще не оцененного пользователя и переход в чат для пользователя, у которого присутствует мэтч с текущим пользователем.

На экране собственного профиля в верхней панели присутствуют кнопки редактирования профиля и настроек, а вместе с постами отображается кнопка создания нового поста. Также в собственном профиле существует возможность отобразить вместо ленты постов список друзей пользователя или его заявок.

В режиме редактирования своего профиля пользователь может изменить имя, описание, город, аватар и фон профиля, а также добавить или удалить фандомы. Изменения сохраняются после подтверждения и отправляются на сервер. При успешном сохранении обновленные данные отображаются в профиле.

В настройках профиля пользователь может сменить свою электронную почту, подтвердив новую почту с помощью ключа верификации, сменить пароль, указать свои предпочтения в отношении уведомлений и приватности, выйти из профиля или удалить его. При выходе из аккаунта активная сессия завершается, и пользователь перенаправляется на начальную страницу приложения. При его удалении происходят аналогичные действия, но при этом учетная запись дополнительно удаляется из базы данных на сервере приложения.


### **Посты**

В приложении присутствует страница ленты, отображающая посты друзей пользователя и его собственные. Она работает аналогично ленте постов в профиле, позволяя ставить постам лайки и открывать экран конкретного поста. При прокрутке страницы реализована подгрузка дополнительных публикаций. Также на этом экране присутствует кнопка перехода на экран нового поста.

Формат поста является одинаковым на всех экранах приложения. Каждый пост содержит аватар автора, его логин, если он доступен, и его имя, а также текст поста, до 5 вложений (изображений и видео), время публикации поста, количество его лайков и комментариев.

Экран поста содержит те же элементы интерфейса, но дополнительно позволяет раскрывать полный текст поста и его список фандомов. Также на экране поста пользователь может просматривать список комментариев к посту и написать свой собственный комментарий. Каждый комментарий содержит имя и аватар автора, время отправки и текст. На экране поста пользователь может написать свой собственный.

Экран создания поста содержит поле для ввода текста поста, кнопку для прикрепления вложений (до 5 изображений и видео суммарно) и поле выбора фандомов. Публикация поста происходит при нажатии соответствующей кнопки в верхней панели.

### **Чаты**

В приложении присутствует функционал личных чатов. На экране списка чатов пользователь может просматривать список своих чатов, фильтровать их по имени собеседника и переходить на экран конкретного чата.

На экране чата пользователь может просматривать историю сообщений и отправлять новые. Сообщения могут содержать текст и до 5 вложений. Чат обновляется в реальном времени. Верхняя панель экрана чата содержит имя и аватар собеседника и позволяет перейти в его профиль.


### **Прочее**

Выбор фандомов возможен только из списка, определяемого серверной частью приложения. При отсутствии нужного пользователю фандома в этом списке он может запросить добавление этого фандома через форму на отдельном экране. Кнопка перехода на этот экран присутствует на всех экранах, где пользователь может добавлять или удалять фандомы (то есть, на экранах фильтров, редактирования профиля и нового поста). Аналогично, выбор города пользователя возможен только из предоставляемого приложением списка.

При получении нового сообщения и при формировании нового мэтча пользователь получает push-уведомления, если до этого он дал разрешение на получение уведомлений и не отключил их через настройки.

На экранах чата и поста есть возможность открыть вложенные изображения и видео в полный экран, а также сохранить их на устройство пользователя.

# **СООБЩЕНИЯ ОПЕРАТОРУ**

## **Уведомления об ошибках**

При работе приложения обрабатываются исключительные ситуации. Сообщения об ошибках поддерживаются в следующих форматах:

### **Базовый экран ошибки**

Данный экран отображается, когда приложению не удалось загрузить с сервера информацию, необходимую для открытия текущей страницы приложения. По нажатию кнопки на экране ошибки происходит повторное обращение к серверу, и при его успехе автоматически осуществляется переход на нужную страницу приложения.

Данный экран ошибки используется на страницах ленты, профиля, списка чатов, чата, поста, редактирования профиля и фильтров. Чаще всего его показ свидетельствует о нестабильном интернет-соединении.


### **Диалог с предупреждением об ошибке**

Данный диалог показывается на экране настроек вместо базового экрана ошибки и используется для отображения критических ошибок, которые требуют внимания пользователя и предлагают выбор действий. В диалоге доступны две кнопки: «Попробовать снова» для повторной попытки запроса и «Отмена» для выхода из текущего сценария.


### **Всплывающее сообщение об ошибке**

Для ошибок, не критичных для текущего сценария использования приложения и не влияющих на состояние текущего экрана, вместо экрана или диалога ошибки показывается небольшое всплывающее сообщение Toast. Оно появляется внизу экрана и информирует пользователя об ошибке.

Toast используется в случае ошибок сохранения фильтров, отправки запроса на добавление фандома, сохранения изменений при редактировании профиля, скачивания медиа в чате или посте, создания нового поста и загрузки аватара во время регистрации.


### **Валидационные сообщения об ошибках**

Валидационные сообщения появляются непосредственно под полем ввода текста при его некорректном значении, описывают, что именно в пользовательском вводе не удовлетворяет условиям данного поля. Также они используются для информирования о сетевых ошибках. Эти сообщения исчезают, когда пользователь исправляет ошибку в поле ввода, или, в случае сетевой ошибки, когда приложение совершает успешный повторный запрос к серверу.

Все поля ввода текста в приложении поддерживают такой формат сообщений об ошибках.


## **Прочие уведомления**

Помимо ошибок, существуют другие случаи, требующие отправки неких сообщений пользователю.

### **Диалог**

Диалог используется при первом запуске приложения для информирования пользователя о том, зачем приложение запрашивает разрешение на отправку push-уведомлений. Также диалог отображается при удалении профиля для уточнения, что данное действие будет необратимо.

### **Всплывающее сообщение**

Всплывающее сообщение Toast показывается в случае успешного окончания процесса загрузки аватара при регистрации, сохранения медиа из поста или чата на устройство, отправки запроса на добавление фандома или восстановления пароля.

### **Push-уведомления**

Push-уведомления системы Android используются для информирования пользователя о событиях, связанных с его профилем: о получении сообщения от другого пользователя или о мэтче с другим пользователем. На экране настроек пользователь может выбрать, какие уведомления он хочет получать.

# **ТЕРМИНЫ И СОКРАЩЕНИЯ**

1. Android Studio — среда разработки для Android-приложений.  
2. Backend — серверная часть приложения.  
3. GitHub — веб\-сервис для хостинга и совместной разработки программного обеспечения, основанный на системе контроля версий Git.  
4. Push-уведомление — сообщение, отправляемое пользователю на устройство.  
5. Категории фандомов — группы тематических направлений (например, «аниме», «фэнтези», «игры», «комиксы»).  
6. Клиент-серверная архитектура — архитектура системы, разбивающая ее на клиентскую и серверную части.  
7. Мэтч  — взаимное выражение симпатии между пользователями, позволяющее начать чат.  
8. Пост — публикация пользователя, содержащая текст, изображение или другой материал.  
9. Пользователь — участник системы, зарегистрировавшийся в приложении.  
10. Профиль — персональная страница пользователя, содержащая имя, фото, описание, список интересов и другие данные.  
11. Рекомендации — список профилей, предлагаемых пользователю для знакомства.  
12. Свайп — жест пролистывания карточек пользователей с оценкой (лайк или дизлайк).  
13. Тег — краткая текстовая метка, описывающая интерес пользователя или его контент.  
14. Фандом — фанатское сообщество, группа поклонников определенного произведения или творческого направления.  
15. Чат — приватная переписка между двумя пользователями после совпадения.
